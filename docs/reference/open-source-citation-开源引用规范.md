# 开源引用规范

这个仓库允许大量吸收开源项目的好思路，但反对两种坏习惯：

- 只写“参考了某项目”，却说不清吸收了什么
- 看了一个项目就机械翻译、机械照搬

## 推荐引用格式

至少写清楚 4 件事：

1. 项目名与链接
2. 吸收了什么方法或机制
3. 没有照搬什么
4. 落地到本仓库的哪个文件或工件

## 推荐表达方式

不要写：

- “参考 gstack”
- “借鉴了 OpenTelemetry”

推荐写成：

- “吸收 gstack 的阶段路由与评审层，但没有照搬它的人设和宿主绑定”
- “吸收 OpenTelemetry 的 trace-log correlation 思路，并落地为 debug-pack 的关联编号与日志聚焦”

## 在哪些地方必须写引用

- README 的参考资料区
- `docs/reference/open-source-inspiration-开源灵感.md`
- commit 正文里有明显外部灵感时
- PR / release note / 设计文档

## 中文化改写要求

如果来源是英文项目或英文概念：

- 先理解原理，再改写成中文工程语言
- 不要把英文原话逐字翻译进主文档
- 要明确哪些表达是“本土化改写后”的工作语言
