# Feature Playbook

Use this for new features, scoped enhancements, and controlled refactors.

## Sequence

1. Generate or refresh the repo map.
2. Create a task packet.
3. Confirm scope, non-goals, constraints, and acceptance criteria.
4. Read the shortlist only.
5. Implement in small steps and keep the trace updated when tradeoffs change.
6. Run validation commands.
7. If the feature closes a previously observed bug class, add an eval case.

## Rules

- Scope belongs in the task packet, not in scattered chat replies.
- Validation commands should be known before editing starts.
- If candidate files balloon beyond the context budget, split the work.
- If implementation crosses domains, stop and write a new packet.
