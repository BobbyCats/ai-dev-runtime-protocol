# Git Commit 提交规范

这个仓库从现在开始，默认要求：

- **提交标题使用中文**
- **写得专业、具体、可审查**
- **不要用模糊词当提交标题**

## 为什么要把这件事写成协议

如果提交信息长期是：

- `update`
- `fix`
- `tweak`
- `misc`

那它对排障、回滚、文档同步、开源协作都几乎没有帮助。

提交信息本质上也是一种运行时索引。

## 标题格式

推荐格式：

`模块: 动作与结果`

例如：

- `设计令牌: 增加 HTML 预览输出`
- `排障协议: 补充 trace-log 关联定位字段`
- `文档同步: 重写 README 的设计令牌章节`

## 标题规则

- 用中文写，不混用无意义的英文动词
- 要写“改了什么”与“为什么有价值”
- 尽量落到模块，不写成笼统的大而空标题
- 不要把验证、背景、参考文献挤进标题

## 正文建议

提交正文推荐包含：

- 背景：为什么要改
- 改动：这次具体做了什么
- 验证：怎么确认改动成立
- 参考：若吸收了开源项目思路，在正文注明

## 文档同步要求

出现这些变化时，提交必须同步体现文档更新：

- CLI 输出变了
- README 的系统定位变了
- 阶段顺序变了
- 新增正式工件或 playbook
