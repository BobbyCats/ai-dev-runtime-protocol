# Feature Playbook | 功能开发手册

适用场景：

- 新功能
- 小范围增强
- 可控范围内的重构

## 标准顺序

1. 先生成或刷新 `repo-map | 仓库地图`
2. 先写 `task-packet | 任务包`
3. 把范围、非目标、约束、验收标准说清楚
4. 只读任务包给出的候选文件
5. 实现过程中如果权衡改变，写入 `decision-trace | 决策轨迹`
6. 跑验证命令
7. 如果顺手解决了一类历史 bug，把它补成回归用例

## 硬规则

- 范围必须落在任务包里，不要散在聊天记录里
- 没写验收标准，不要直接开工
- 候选文件一旦超出上下文预算，应该拆任务而不是硬顶
- 一旦跨域，重新写任务包
