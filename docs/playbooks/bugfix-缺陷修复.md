# Bugfix Playbook | 缺陷修复手册

适用场景：

- 用户反馈 bug
- 测试突然失败
- 线上行为和预期不一致

## 标准顺序

1. 先生成或刷新 `repo-map | 仓库地图`
2. 再生成 `debug-pack | 排障包`
3. 先复现，再改代码
4. 先读排障包里的短名单，不要一上来全仓扫描
5. 关键判断变化时，写入 `decision-trace | 决策轨迹`
6. 只修最能解释问题的那一层
7. 把这个 bug 变成 `eval-case | 回归用例`
8. 跑验证命令

## 硬规则

- 没有先复现，就不要大范围改逻辑
- 没有排障包，就不要默认全仓搜索
- 没有证据，不要凭感觉扩大范围
- 对真实用户有影响的 bug，修完必须留下回归资产

## 最少需要的工件

- `repo-map`
- `debug-pack`
- `decision-trace`
- `eval-case`
