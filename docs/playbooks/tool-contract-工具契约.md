# 工具契约（Tool Contract）

适用场景：

- Agent 已经开始调多个工具
- 你发现工具越写越聪明、越写越黑盒
- 你需要把“Smart Agent, Dumb Tools”落实成工程边界

## 这一步解决什么问题

如果工具同时负责：

- 解析
- 校验
- 业务判断
- 执行
- 写库

那 bug 一来，你几乎不知道问题是：

- Agent 判断错了
- 工具契约不清楚
- 工具内部偷偷改了规则

## 一份好契约至少要写清楚

- 工具用途
- 输入字段与类型
- 输出字段与类型
- 幂等规则
- 权限边界
- 失败语义
- 是否允许自动重试
- 是否允许回滚

## 推荐规则

- 工具只负责执行，不偷偷帮你做需求理解
- 高风险校验优先用确定性规则，不要只靠 LLM
- 工具返回必须结构化，不能只给一段自然语言
- 失败码要稳定，方便排障和回归
